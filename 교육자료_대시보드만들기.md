# AI 대시보드 제작 교육 가이드

## 개요
- 코딩 지식 없이 AI(Kiro)와 대화하며 웹 대시보드를 만드는 방법
- 소요 시간: 환경세팅 30분 + 제작 2~3시간 + 배포 10분

---

## STEP 1. 환경 세팅 (최초 1회)

### 1-1. 필요한 프로그램 설치

| 프로그램 | 용도 | 설치 주소 |
|---------|------|----------|
| Kiro | AI 에디터 (코드 작성) | AWS 계정으로 가입 |
| Git | 버전 관리 및 배포 | https://git-scm.com/download/win |
| GitHub 계정 | 코드 저장소 + 웹 호스팅 | https://github.com |

### 1-2. Git 초기 설정 (설치 후 1회)

Kiro 터미널(Ctrl + `)에서 실행:
```
git config --global user.name "본인깃허브아이디"
git config --global user.email "본인이메일@example.com"
```

### 1-3. 작업 폴더 만들기

바탕화면에 폴더 생성 (예: `my-dashboard`)
Kiro에서 File > Open Folder로 해당 폴더 열기

---

## STEP 2. AI와 대화해서 만들기

### 2-1. 첫 프롬프트 작성법 (가장 중요)

첫 메시지에 전체 스펙을 한번에 줘야 합니다.

좋은 예시:
```
웹 대시보드를 만들어줘.
- 왼쪽: 지도 이미지 업로드 + 편집 캔버스
- 오른쪽: 데이터 입력 패널
- 기능: 지도 위에 영역 그리기, 데이터 입력, PPT 내보내기
- 스타일: 다크모드, 깔끔한 UI
```

나쁜 예시:
```
뭔가 멋진거 만들어줘
```

### 2-2. 수정 요청 잘하는 법

핵심 규칙: **구체적으로 말할수록 정확하게 나온다**

| 안 좋은 요청 | 좋은 요청 |
|-------------|----------|
| 글씨 좀 키워줘 | 격자 안 글씨를 30pt로 키워줘 |
| 색 바꿔줘 | 주동선 색상을 rgb(255,0,0) 투명도 45%로 변경 |
| 위치가 이상해 | 지도를 위에서 1.5cm, 왼쪽에서 0.75cm 위치로 옮겨줘 |
| 안됨 | 배율 입력하고 엔터 눌러도 반영 안됨 |

### 2-3. 여러 개 한번에 요청

번호 매겨서 한번에 던지면 효율적:
```
1. 격자를 4x4로 변경
2. 글씨 크기 30pt
3. 동선 색상 빨강으로
4. PPT 표 간격 0.3cm
```

### 2-4. 결과 확인 방법

Kiro 터미널에서:
```
start index.html
```
브라우저에서 열림. 수정 후 확인할 때는 브라우저에서 Ctrl+Shift+R (강력 새로고침)

### 2-5. 문제 생겼을 때

- 화면이 안 바뀜 → 브라우저 강력 새로고침 (Ctrl+Shift+R)
- 코드가 깨짐 → "전체 재작성해줘" 요청
- 기능이 안됨 → 뭐가 안되는지 구체적으로 설명 + 스크린샷 첨부

---

## STEP 3. 배포 (깃허브에 올리기)

### 3-1. GitHub 레포 생성

1. https://github.com 로그인
2. 우측 상단 + 버튼 > New repository
3. 이름 입력 (예: my-dashboard)
4. Public 선택
5. Create repository 클릭

### 3-2. 코드 올리기

Kiro 터미널에서 (최초 1회):
```
git init
git add .
git commit -m "첫 버전"
git branch -M main
git remote add origin https://github.com/본인아이디/레포이름.git
git push -u origin main
```

### 3-3. 웹사이트 켜기 (GitHub Pages)

1. GitHub 레포 페이지 > Settings
2. 왼쪽 메뉴 Pages
3. Branch: main / root 선택 > Save
4. 1~2분 후 접속 가능: https://본인아이디.github.io/레포이름/

### 3-4. 수정 후 업데이트

수정할 때마다 이 3줄만 실행:
```
git add .
git commit -m "수정 내용 설명"
git push
```

## 자주 쓰는 터미널 명령어 치트시트

| 명령어 | 용도 |
|--------|------|
| start index.html | 브라우저에서 열기 |
| git add . | 변경된 파일 준비 |
| git commit -m "메시지" | 변경사항 저장 |
| git push | GitHub에 업로드 |
| git status | 현재 상태 확인 |

## 프롬프트 템플릿 모음

### 대시보드 초기 생성
```
[프로젝트명] 웹 대시보드를 만들어줘.
순수 HTML/CSS/JS로 서버 없이 브라우저에서 동작.

구조:
- 왼쪽: [캔버스/지도/차트 영역]
- 오른쪽: [데이터 입력 패널]

기능:
1. [기능1]
2. [기능2]
3. [기능3]

스타일: [다크모드/밝은톤], [폰트], [색상 톤]
```

### 수정 요청
```
다음을 수정해줘:
1. [구체적 위치] + [구체적 변경 내용] + [수치/색상]
2. ...
```

### PPT 내보내기 추가
```
PPT 내보내기 기능을 추가해줘.
- 슬라이드 크기: 와이드스크린 (33.867cm x 19.05cm)
- 왼쪽: [이미지 영역]
- 오른쪽: [표 영역, 가로 Xcm]
- 폰트: 맑은고딕
- 표 테두리: 검정
```

## 팁

- AI는 한국어로 말해도 잘 알아들음
- 이미지를 첨부해서 "이 부분 이상해" 하면 더 빠름
- 한 대화에서 계속 수정하면 맥락을 기억함
- 너무 길어지면 "전체 재작성해줘" 하고 새로 시작하는게 깔끔
- 코드 몰라도 됨. 결과물만 보고 피드백하면 됨
